# Linux Services and systemctl

## Objective
Learn how to manage system services using systemctl on a Red Hat–based Linux system.

## Environment
- Linux (Red Hat–based distribution)
- systemd
- Terminal / command line

## Commands Practiced
- systemctl status
- systemctl start
- systemctl stop
- systemctl restart
- systemctl enable
- systemctl disable
- systemctl is-active
- systemctl is-enabled
- systemctl list-units --type=service

## What I Did
- Checked the status of running services
- Started and stopped services
- Restarted services after configuration changes
- Enabled and disabled services at boot
- Verified service states using systemctl queries

## Security & Operations Relevance
- Ensured critical services are running
- Disabled unnecessary services to reduce attack surface
- Practiced troubleshooting service failures

## Outcome
Built confidence managing Linux services in an enterprise environment using systemctl.
